## VS(Version control System)

- 파일변화를 시간에 따라 기록했다가 특정 시점의 버전을 다시 꺼내올 수 있게 해주는 시스템이다.

### VS의 장점

- 변경된 내용을 공유하기에 용이하다.
- 파일 이름을 변경하지 않고 버전관리가 가능하다.
- 파일을 원하는 시점으로 복구하기 쉽다.
- branch를 만들어 병렬 관리가 가능하다.
- 타인과 함꼐 작업한 내용을 병합하기 쉽다.

## Git의 작업흐름

- 깃에는 총3가지 작업환경이 있다. (working directory, staging area, .git directory)
- working directory: 프로젝트의 파일을 작업하는 공간이다.
- staging area: 버전을 만들 준비가 되어있는 파일을 옮겨 놓는 곳이다. add 명령어를 사용하여 파일을 stageArea로 옮길 수 있다.
- .git directory: 만들어진 버전이 저장되는 곳이다. add 명령어로 staged된 파일을 commit 명령어를 사용하여 버전을 만들 수 있다.

## Git 명령어

- cd 해당 디렉토리로 이동한다.
- cd .. 부모 디렉토리로 이동한다.
- mkdir: 새 디렉토리를 만든다.
- pwd: 디렉토리의 경로를 확인한다.
- clear: 터미널의 모든 내용을 지운다.
- git init: 버전 관리를 시작하려는 디렉토리를 git에게 알려준다.
- touch 파일명: 빈 파일을 생성한다.
- ls: 디렉토리 내의 파일 목록을 보여준다.
- ls -al: 디렉토리 내의 파일 목록을 보준다.(숨겨진 파일까지 전부)
- git status: 커밋되지 않은 변경사항을 조회한다.
- git add: 파일을 stageArea로 옮기고, Git이 파일을 추적하도록 한다.
- git commit -am "메세지" : add를 생략하고 바로 커밋을 한다.(한번이상 add한 파일에만 사용가능)
- git commit -m "메세지"
- git commit: 버전을 만든다.
- git log: 커밋 로그를 확인한다.
- git reset
    - git reset --hard 커밋ID: hard옵션을 적용하면 해당 커밋ID의 상태로 워킹디렉토리와 Index영역 모두 초기화 된다. 또한 변경 이력 및 내용이 전부 삭제된다.(HEAD, Index, working directory reset)
    - git reset --mixed: reset의 기본값이며, 커밋ID의 상태로 Index영역은 초기화되고 working directory는 변경 되지 않는다. 변경 이력 및 내용은 그대로 남아있다.
    - git reset --soft  ???..
    - git reset HEAD 파일명: add 취소
- git branch: 브랜치 목록 보여준다.
- git branch -d 브랜치명: 해당 브랜치를 삭제한다.
- git checkout 브랜치명: 해당 브랜치로 이동한다.
- git checkout -b 브랜치명: 브랜치를 만들고 checkout
- git stash save: 현재 브랜치에서 작업 도중에 다른 브랜치로 이동해야할떄 사용. 현재 작업 중인 내용을 임시로 저장하고 워킹디렉토리를 비운다.
- git stash apply:가장 최근에 stash 한 작업을 복구한다.
- git stash list: stash 한 목록을 보여준다.(가장 위에 작업이 최신)
- git stash drop: 가장 최근에 stash 한 작업을 삭제한다.
- git stash pop: git apply and git drop
- git remote add origin 원격저장소경로:원격저장소를 연결한다.
- git remote -v: 원격저장소의 단축이름과 URL을 함께 보여준다.
- git remote remove origin:원격저장소의 연결을 해제한다.
- git push 저장소명 브랜치명: 원격 저장소에 커밋한 내용을 업로드한다.

## HEAD

HEAD는 현재 브랜치를 가리키는 포인터이며, 브랜치는 브랜치에 담긴 커밋 중 가장 마지막 커밋을 가리킨다. 지금의 HEAD가 가리키는 커밋은 바로 다음 커밋의 부모가 된다.
